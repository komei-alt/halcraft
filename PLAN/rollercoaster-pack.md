# ジェットコースターパック

## 概要

- **目的**: プレイヤーがレールブロックを自由に設置し、高低差を活用した任意形状のジェットコースターコースを作成・走行できる機能
- **非目標**: マルチプレイ同期（初期リリースでは除外）、レール自動生成

## 設計方針の比較

### 案A: レールブロック方式（推奨）
- **何が変わるか**: 新しい「レールブロック」をブロックシステムに追加。プレイヤーが1ブロックずつ設置して自由にコースを構築する
- **なぜそれで足りるか**: 既存のブロック設置/破壊システムをそのまま活用できる。マイクラの線路と同じ直感的な操作
- **主なリスク**: カーブ表現が6方向（NSEW+上下）に限定される → 斜め方向の補助ブロックで対応可能

### 案B: フリードロー方式
- **何が変わるか**: 専用エディタUIで3D空間にベジェ曲線でコースを描く
- **なぜそれで足りるか**: 滑らかなカーブを自由に作れる
- **主なリスク**: UI実装が大幅に重い。ブロックワールドの世界観から外れる

### 案C: プリセットパーツ方式
- **何が変わるか**: 直線/カーブ/ループなどの定形パーツを連結する
- **なぜそれで足りるか**: パーツごとに物理が予測しやすい
- **主なリスク**: 自由度がブロック方式より低い。パーツ種が増えると管理コスト増

→ **推奨: 案A（レールブロック方式）** — ブロックワールドの世界観に最もフィットし、既存システムとの統合が容易

## アーキテクチャ

### レールブロック体系

```
RAIL_STRAIGHT (25) — 直線レール（N-S / E-W 自動判定）
RAIL_CURVE    (26) — 90°カーブレール（設置方向で回転）
RAIL_SLOPE_UP (27) — 上昇レール（1ブロックあたり45°上昇）
RAIL_SLOPE_DN (28) — 下降レール（1ブロックあたり45°下降）
```

各レールブロックは `nonStandard: true`, `noCollision: true` でプレイヤーは通過可能。
レール上に車体を設置して搭乗するとコースター物理に切り替わる。

### コース検出アルゴリズム

```
1. 車体の現在位置からレールブロックを検索
2. 隣接レールを方向ベースで探索し、コースグラフを構築
3. グラフ上の位置をパラメトリックに管理（0.0〜1.0 per block）
4. 次のブロックへの遷移は境界到達時に自動実行
```

### 物理エンジン

```typescript
// コースター物理の核心
interface CoasterPhysics {
  /** レール上のパラメトリック位置 */
  t: number;
  /** 現在速度（m/s） */
  speed: number;
  /** 現在のレールブロック座標 */
  currentRail: { x: number; y: number; z: number };
  /** 進行方向（レールの向き） */
  direction: RailDirection;
}

// 毎フレームの速度更新
function updateCoasterSpeed(physics: CoasterPhysics, dt: number): void {
  const GRAVITY = 25;           // 既存のプレイヤー重力と一致
  const FRICTION = 0.02;        // レール摩擦係数
  const MAX_SPEED = 30;         // 最高速度（m/s）

  // レールの勾配角度からsin値を取得
  const slopeAngle = getRailSlopeAngle(physics.currentRail);

  // 重力による加速: a = g * sin(θ)
  const gravityAccel = GRAVITY * Math.sin(slopeAngle);

  // 摩擦による減速
  const frictionDecel = FRICTION * physics.speed * physics.speed * Math.sign(physics.speed);

  // 速度更新
  physics.speed += (gravityAccel - frictionDecel) * dt;
  physics.speed = clamp(physics.speed, -MAX_SPEED, MAX_SPEED);
}
```

### レール方向と接続ロジック

```
方向定数:
  NORTH (+Z) / SOUTH (-Z) / EAST (+X) / WEST (-X) / UP (+Y) / DOWN (-Y)

接続ルール:
  直線: 進行方向の前後のみ接続
  カーブ: 入口方向と出口方向（90°回転）の2方向に接続
  上昇: 水平方向の入口 → 1ブロック上の出口
  下降: 1ブロック上の入口 → 水平方向の出口

隣接探索:
  現在ブロックの出口方向にあるブロックがレールかチェック
  レールなら接続方向が合致するか検証
  合致すれば遷移、なければ停止（終端）
```

## 画面マップ

### レール設置（既存フロー）
```
ホットバー → レールブロック選択 → 右クリックで設置 → 隣接レール自動接続表示
```

### コースター搭乗
```
車体ブロック付近で F キー → 搭乗 → 三人称カメラ（車体後方）
→ Space で発進 / ブレーキ
→ 物理演算でレール上を走行
→ F キーで降車
```

## リスク・保留事項

### 判断が必要な点

1. **レールのカーブ表現**: 90°カーブのみか、45°の緩いカーブも追加するか
   - → 初期は90°のみで実装。需要があれば後から追加

2. **ループ（宙返り）**: 360°ループを物理的に正しくサポートするか
   - → Phase 1 では除外。速度が足りないとループ途中で落下するロジックは複雑

3. **車体のクラフトコスト**: 鉄6 + 木2 程度を想定だが、バランス調整が必要
   - → 仮値で実装し、プレイテストで調整

4. **マルチプレイ対応**: 車体の位置同期、同乗者サポート
   - → Phase 2以降で `useMultiplayerStore` に統合

### 技術リスク

- **パフォーマンス**: 長大なコースでレール数が数百を超える場合の描画負荷
  - → InstancedMesh + チャンク単位描画で対応（World.tsx と同じパターン）

- **カーブ補間**: ブロック境界でのカクつき
  - → Catmull-Rom スプライン補間でブロック間を滑らかに接続
